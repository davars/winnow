# Plan: `winnow organize` subcommand

## Context

Winnow currently walks, hashes, and enriches (MIME, EXIF) files in raw/, then applies rules (junk, dedup) that move files to clean/ or trash/. There is no component that actually *organizes* media into a date-structured layout in `clean/`.

This plan adds a new standalone `winnow organize` subcommand that copies image and video files from `raw` to `clean/media/YYYY/MM/<name>.<ext>` based on the best available capture timestamp. An optional `--remove-originals` flag also deletes the source from `raw`, effectively turning the copy into a move.

Copy-by-default is deliberate (different semantic from the existing rule/plan/process flow, which always moves) — it lets the user organize safely, verify the result, then remove originals in a second pass if desired.

## Design Decisions (from user)

| Decision | Choice |
|---|---|
| Integration | Standalone subcommand (`winnow organize`), not a rule |
| Filename | `<ISO-UTC>-<short-sha8>-<basename>.<ext>` |
| Naive EXIF TZ | `EXIF:OffsetTimeOriginal`/`OffsetTime` when present, else **explicit config `timezone`** (no system-local fallback) |
| Idempotency | SHA256 check in `clean` store + O_EXCL on open |

Filename example: `20181030T211730Z-a1b2c3d4-DSC_0123.jpg`
Full path example: `clean/media/2018/10/20181030T211730Z-a1b2c3d4-DSC_0123.jpg`

## Config: Required `timezone` Field

A new `[organize] timezone = "..."` TOML field (IANA name, e.g. `"America/New_York"`). Required for `winnow organize` to run — rejected if empty or un-loadable by `time.LoadLocation`. Other subcommands ignore it.

Rationale: naive EXIF timestamps must be interpreted deterministically; using system-local TZ silently makes the year/month folder depend on where winnow runs (not reproducible across machines/containers). Making it explicit forces the user to decide once and documents the assumption.

**`config/config.go` changes**:
- Add `OrganizeConfig struct { Timezone string \`toml:"timezone"\` }`.
- Add `Organize OrganizeConfig \`toml:"organize"\`` to `Config`.
- Add method `func (c *Config) Location() (*time.Location, error)` that returns `time.LoadLocation(c.Organize.Timezone)` with a helpful error ("organize.timezone not set — run `winnow init` or edit config") when empty.
- `Validate()` does **not** check timezone (it's only required for `organize`, not e.g. `walk`).

**`cmd/init.go` + new `cmd/tzprompt.go`**: Add a `pickTimezone(reader, out)` helper with fuzzy matching so users don't need to remember exact IANA casing.

Flow:
1. Suggest a default by reading `/etc/localtime` symlink (e.g., `/var/db/timezone/zoneinfo/America/Los_Angeles` → `"America/Los_Angeles"`). Falls back to `"UTC"` if detection fails or target isn't under a zoneinfo path.
2. Load the list of valid IANA zone names by walking `/usr/share/zoneinfo/` and `/var/db/timezone/zoneinfo/` (macOS). Skip known non-zone entries (`posix/`, `right/`, `.tab` files, `leapseconds`, files starting with lowercase ASCII letter at top level like `zone1970.tab`). Deduplicate across roots. On error (e.g., Windows dev machines), fall back to plain validation — no fuzzy list.
3. Prompt: `Timezone [America/Los_Angeles]: `
4. On empty input: accept the default.
5. On exact match (case-sensitive) to a known zone: accept.
6. Otherwise: fuzzy match. Normalize query and zone names by lowercasing and replacing `_`, `/`, `-` with space; split query on whitespace into tokens; a zone matches if every token is a substring of the normalized zone name. Rank: exact-case-insensitive > starts-with > contains.
   - 1 match → show `Using: America/New_York` and accept.
   - 2–10 matches → print numbered list; user types a number or re-queries.
   - >10 matches → show first 10 with `… and N more, be more specific` and re-prompt.
   - 0 matches → `No matches. Try a region (e.g. "europe") or exact IANA name.` and re-prompt.
7. Final guard: `time.LoadLocation` on the selected value; reject `"Local"` and `""`.

Function signatures in `cmd/tzprompt.go`:
```go
func pickTimezone(reader *bufio.Reader, defaultTZ string) (string, error)
func detectSystemTZ() string                        // reads /etc/localtime symlink
func loadZoneList() ([]string, error)               // walks zoneinfo dirs
func matchZones(query string, zones []string) []string // fuzzy, ranked
func normalizeZone(s string) string                 // lowercase, separators -> space
```

**`cmd/organize.go`**: Calls `cfg.Location()` at the start of `runOrganize`; a non-nil error aborts before any DB/FS work.

## Timestamp Priority

Parsed from `exif.data` JSON in this order; first non-empty wins:

1. `Composite:SubSecDateTimeOriginal`
2. `EXIF:DateTimeOriginal` (+ `EXIF:OffsetTimeOriginal` if naive)
3. `EXIF:CreateDate` (+ `EXIF:OffsetTime` if naive)
4. `QuickTime:CreateDate`
5. `XMP:CreateDate`
6. `RIFF:DateTimeOriginal`
7. Fallback: `files.mod_time` (already ISO UTC)

Formats handled: `YYYY:MM:DD HH:MM:SS`, `YYYY:MM:DD HH:MM:SS.mmm`, with optional trailing `Z` or `±HH:MM`. Naive values without a matching offset tag are parsed in the configured `organize.timezone` and converted to UTC for the filename.

The `operations.reason` column records which tag was used (e.g., `"source: EXIF:DateTimeOriginal"`) so later SQL sweeps can re-organize mod_time-sourced files if capture metadata is recovered.

## SQL

**Precondition** (fail early if MIME enricher hasn't run):
```sql
SELECT COUNT(*) FROM mime
```

**Candidate fetch**:
```sql
SELECT f.id, f.sha256, f.path, f.size, f.mod_time, e.data
FROM files f
JOIN mime m ON m.hash = f.sha256
LEFT JOIN exif e ON e.hash = f.sha256
WHERE f.store = 'raw' AND f.missing = 0 AND f.sha256 IS NOT NULL
  AND m.mime_type IN (?, ?, ...)  -- enricher.EXIFMimeTypes
ORDER BY f.path
```

**Idempotency**:
```sql
SELECT 1 FROM files WHERE store='clean' AND sha256=? AND missing=0 LIMIT 1
```

**Per-file DB tx** (after successful copy):
```sql
INSERT INTO files (store, path, size, mod_time, found_at, reconciled_at, missing, sha256, hashed_at)
VALUES ('clean', ?, ?, ?, ?, ?, 0, ?, ?);

-- Only if --remove-originals and os.Remove succeeded:
DELETE FROM files WHERE id = ?;  -- raw row id

INSERT INTO operations (file_id, src_store, src_path, dst_store, dst_path, rule, reason, executed_at)
VALUES (?, 'raw', ?, 'clean', ?, 'organize', ?, ?);  -- file_id = new clean row id
```

## Per-File Ordering

1. Parse capture time (in memory).
2. Build dest path.
3. Query `files` for existing clean-store SHA256 match → skip if found.
4. Dry-run short-circuit: print and continue.
5. `copyFile(srcAbs, dstAbs)` — MkdirAll, O_EXCL open; on EEXIST return sentinel `errDstCollision`, count as collision and skip.
6. If `--remove-originals`: `os.Remove(srcAbs)`; failure logs to `process_errors`, falls back to copy-only (do not delete raw row).
7. Open per-file DB tx: INSERT clean row, conditionally DELETE raw row, INSERT operations row, commit.
8. Per-file errors → `process_errors` (matches `plan.go:368-381` pattern); sequential loop continues.

Per-file transactions (not batched) because each write depends on the FS outcome and the new row's `LastInsertId()`. Scale is thousands of files, not millions — acceptable.
